> **RouteOptimizer** — учебное демонстрационное WPF-приложение для работы с маршрутной сетью, поиском маршрутов и базовой аналитикой.  
> Проект **не является готовым production-решением** и сделан как наглядная реализация под курсовую работу.

---

## О проекте

Приложение подключается к существующей базе данных **MS SQL Server** и демонстрирует типовой набор функций для системы управления городским транспортом:

- авторизация по логину и паролю;
- ролевой доступ к функциям;
- просмотр и редактирование маршрутов;
- управление остановками маршрута;
- ввод пассажиропотока;
- поиск маршрута между остановками через хранимую процедуру;
- простая аналитика по маршрутам;
- моделирование активности маршрутов и остановок.

---

## Поддерживаемые роли

### Editor
Полный доступ к административным операциям:
- список маршрутов;
- добавление / редактирование / удаление маршрутов;
- управление остановками маршрута;
- редактирование пассажиропотока.

### Planner
Планировщик видит:
- поиск маршрута;
- аналитические данные;
- моделирование активности маршрутов.

### Guest
Гость видит только:
- поиск маршрута между остановками.

---

## Функциональность

### 1. Авторизация
- логин и пароль;
- получение роли пользователя из таблицы `Users` и `Roles`;
- показ/скрытие вкладок в зависимости от роли.

### 2. Маршруты
- таблица маршрутов;
- поля: номер, тип транспорта, оператор, интервал, время начала/окончания, активность;
- создание, редактирование и удаление маршрута;
- подтверждение удаления.

### 3. Остановки маршрута
- выбор маршрута;
- список остановок с порядком;
- добавление остановки из общего списка;
- удаление;
- перемещение вверх/вниз;
- пересчёт порядка после удаления.

### 4. Пассажиропоток
- выбор маршрута и перегона;
- ввод значений по периодам;
- сохранение в `PassengerFlow`.

### 5. Поиск маршрута
- выбор начальной и конечной остановки;
- вызов `sp_FindRoutesBetweenStops` через `FromSqlRaw`;
- вывод результатов в `DataGrid`.

### 6. Аналитика
- суммарная нагрузка по маршрутам;
- список пересекающихся / дублирующих участков;
- простая табличная визуализация.

### 7. Моделирование
- переключение активности маршрутов и остановок;
- запись флага `is_active` в БД;
- базовая проверка логики перед отключением.

## Запуск

### 1. Настройка строки подключения
Откройте `appsettings.json` и укажите свой сервер SQL Server:

```json
{
  "ConnectionStrings": {
    "DefaultConnection": "Server=YOUR_SERVER;Database=RouteOptimumDB;Trusted_Connection=True;TrustServerCertificate=True;"
  }
}
```

### 2. Запуск из Visual Studio
- открыть решение;
- восстановить NuGet-пакеты;
- собрать проект;
- запустить `RouteOptimizer`.

### 3. Публикация в EXE
Для сборки portable-версии:

```bash
dotnet publish -c Release -r win-x64 --self-contained true -p:PublishSingleFile=true -p:IncludeNativeLibrariesForSelfExtract=true -o ./publish
```

После публикации приложение можно запускать на Windows 10/11 без установленного .NET Runtime.
